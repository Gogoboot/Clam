## 🗓 Рекомендуемый порядок

```
1. Drag & Drop              — быстро, сразу заметно
2. useMemo + useCallback    — быстро, улучшает производительность
3. Разделение на хуки       — упростит дальнейшую разработку
4. Разделение на компоненты — после хуков станет легче
5. Ползунок яркости         — быстрая победа по UI
6. Скачивание моделей       — убирает ручную установку
7. Мультипровайдер LLM      — расширяет аудиторию
8. Очередь файлов           — сложно, но очень полезно
9. История транскрипций     — долгосрочная ценность
10. Встроенный плеер        — завершает основной функционал
11. Диаризация              — продвинутая фича
```
